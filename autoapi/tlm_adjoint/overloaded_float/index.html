tlm_adjoint.overloaded_float

This module defines types which allow for basic floating point level algorithmic differentiation. The implementation is intended to be used for a small number of calculations, for example after the calculation of a functional obtained from a finite element model.

For example, if annotation and operator overloading is enabled

import numpy as np

x = Float(np.pi, name='x')
y = x * np.sin(x)

will lead to annotation of equations associated with the floating point calculations.

class tlm_adjoint.overloaded_float.SymbolicFloat(value=0.0, *, name=None, space_type='primal', static=False, cache=None, checkpoint=None, dtype=None, comm=None, annotate=None, tlm=None)

A SymPy Symbol which is also a ‘function’, defining a scalar variable.

If constructing SymPy expressions then the SymbolicFloat class should be used instead of the OverloadedFloat subclass, or else OverloadedFloat operator overloading should be disabled.

Parameters

  • value – A scalar or SymPy Expr defining the initial value. If a SymPy Expr then, if annotation or derivation and solution of tangent-linear equations is enabled, an assignment is processed by the tlm_adjoint.tlm_adjoint.EquationManager manager.

  • name – A str name for the SymbolicFloat.

  • space_type – The space type for the SymbolicFloat. ‘primal’, ‘dual’, ‘conjugate’, or ‘conjugate_dual’.

  • static – Defines the default value for cache and checkpoint.

  • cache – Defines whether results involving this SymbolicFloat may be cached. Default static.

  • checkpoint – Defines whether a tlm_adjoint.checkpointing.CheckpointStorage should store this SymbolicFloat by value (checkpoint=True) or reference (checkpoint=False). Default not static.

  • dtype – The data type associated with the SymbolicFloat. Typically numpy.float64 or numpy.complex128. Defaults to numpy.complex128.

  • comm – The mpi4py.MPI.Comm associated with the SymbolicFloat.

  • annotate – Whether the tlm_adjoint.tlm_adjoint.EquationManager should record the solution of equations.

  • tlm – Whether tangent-linear equations should be solved.

class tlm_adjoint.overloaded_float.FloatEquation(x, expr)

Represents an assignment to a SymbolicFloat x,

\[x = \mathcal{G} \left( y_1, y_2, \ldots \right),\]

for some \(\mathcal{G}\) defined by a SymPy Expr. The forward residual is defined

\[\mathcal{F} \left( x, y_1, y_2, \ldots \right) = x - \mathcal{G} \left( y_1, y_2, \ldots \right).\]
Parameters

  • x – A SymbolicFloat defining the forward solution \(x\)

  • expr – A SymPy Expr defining the right-hand-side.

forward_solve(self, x, deps=None)

Compute the forward solution.

Can assume that the currently active tlm_adjoint.tlm_adjoint.EquationManager is paused.

Parameters

  • X – A function if the forward solution has a single component, otherwise a Sequence of functions. May define an initial guess, and should be set by this method. Subclasses may replace this argument with x if the forward solution has a single component.

  • deps – A tuple of functions, defining values of dependencies. Only the elements corresponding to X may be modified. self.dependencies() should be used if not supplied.

adjoint_jacobian_solve(self, adj_x, nl_deps, b)

Compute an adjoint solution.

Parameters

  • adj_X – Either None, or a function (if the adjoint solution has a single component) or Sequence of functions (otherwise) defining the initial guess for an iterative solve. May be modified or returned. Subclasses may replace this argument with adj_x if the adjoint solution has a single component.

  • nl_deps – A Sequence of functions defining values of non-linear dependencies. Should not be modified.

  • B – The right-hand-side. A function (if the adjoint solution has a single component) or Sequence of functions (otherwise) storing the value of the right-hand-side. May be modified or returned. Subclasses may replace this argument with b if the adjoint solution has a single component.

Returns

A function or Sequence of functions storing the value of the adjoint solution. May return None to indicate a value of zero.

subtract_adjoint_derivative_actions(self, adj_x, nl_deps, dep_Bs)

Subtract terms from other adjoint right-hand-sides.

Can be overridden for an optimized implementation, but otherwise uses tlm_adjoint.equation.Equation.adjoint_derivative_action().

Parameters

  • adj_X – The adjoint solution. A function if the adjoint solution has a single component, otherwise a Sequence of functions. Should not be modified. Subclasses may replace this argument with adj_x if the adjoint solution has a single component.

  • nl_deps – A Sequence of functions defining values of non-linear dependencies. Should not be modified.

  • dep_Bs – A Mapping whose items are (dep_index, dep_B). Each dep_B is a tlm_adjoint.adjoint.AdjointRHS which should be updated by subtracting derivative information computed by differentiating with respect to self.dependencies()[dep_index].
